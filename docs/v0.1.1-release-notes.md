# KeyScout v0.1.1-alpha Release Notes

KeyScout v0.1.1-alpha refreshes the first alpha with clearer install guidance,
better Accessibility permission recovery, and a release pipeline that is ready
for future Developer ID signed and notarized artifacts.

## Highlights

- Clearer README and development docs for the three supported run modes:
  release download, local `.app` bundle, and `swift run KeyScout`.
- More specific missing Accessibility permission text in the menu.
- `Reveal KeyScout in Finder` menu action so testers can find the exact running
  app bundle to grant Accessibility permission to.
- Developer ID signing and notarization-capable packaging scripts.
- GitHub `App Artifact` workflow support for signed/notarized artifacts when
  maintainer secrets are configured.
- Release docs now explain that GitHub Actions secrets are private and should
  only be exposed to trusted release workflows.

## Requirements

- macOS 14 or newer

## Download

Download `KeyScout.app.zip` and verify it with `KeyScout.app.zip.sha256`.

For the alpha, unzip `KeyScout.app.zip`, preferably move `KeyScout.app` to
`/Applications`, and grant Accessibility permission to that exact app. If
permission gets confused, remove old KeyScout entries from System Settings >
Privacy & Security > Accessibility, then add the exact app again.

Contributors should use `scripts/build_app.sh && open dist/KeyScout.app` for
Accessibility testing, not `swift run KeyScout`.

## Signing Status

This alpha artifact is ad-hoc signed, not Developer ID signed, and not
notarized. macOS Gatekeeper warnings are expected.

The repository can now produce Developer ID signed and notarized artifacts once
Apple Developer certificate and notary secrets are configured, but this release
does not include those credentials.
