# KeyScout v0.1 Release Notes Draft

KeyScout v0.1 has not been released yet. This draft should be updated with real
QA results before publishing.

## Highlights

- Native macOS menu bar app scaffold
- Best-effort frontmost app shortcut scanning through Accessibility APIs
- Accessibility permission status in the menu with a settings shortcut
- Searchable shortcut list window with source filtering and selected-row
  conflict details
- JSON import for curated/manual mapping files
- Persistent local storage for imported mappings
- Deterministic unused shortcut suggestion
- Shortcut conflict lookup
- JSON export of the latest shortcut catalog
- Local-only behavior with no account, analytics, telemetry, or cloud sync
- Ad-hoc signed zipped app artifact with SHA-256 checksum for maintainer testing

## Requirements

- macOS 14 or newer

## Known Limitations

- Accessibility scanning is best-effort and currently limited to frontmost app
  menus.
- Global hotkeys registered outside normal app menus are not detected yet.
- The JSON format is documented but does not include a schema version yet.
- Builds are ad-hoc signed, not Developer ID signed, and not notarized unless
  release automation changes before v0.1.
- macOS Gatekeeper warnings are expected for non-notarized artifacts.

## Verification

- Automated checks on 2026-06-04: `swift test` passed with 22 Swift Testing
  tests; `scripts/build_app.sh` built `dist/KeyScout.app`.
- Manual app checks on 2026-06-04: launch, menu bar item, missing Accessibility
  permission status, settings shortcut, shortcut list window, JSON export, valid
  JSON import, invalid JSON import error status, and local-only behavior passed.
- Accessibility QA on 2026-06-04: Finder scanned 108 shortcuts, Safari scanned
  93 shortcuts, list rows populated, search/source filtering worked, selected-row
  conflict detail worked, JSON export wrote 108 Finder shortcuts, and imported
  mappings persisted after restart.
- Import caveat: a valid import reports success, but imported mappings do not
  appear in the shortcut list until they match a scanned app name or bundle
  identifier.
- Frontmost-app caveat: generating while KeyScout's own list window is frontmost
  can replace the latest catalog with an empty/non-useful scan.
