# v0.1 Release Checklist

KeyScout has not reached v0.1 yet. Use this checklist before tagging or
publishing the first alpha/beta release.

## Automated Checks

- [ ] `swift build`
- [ ] `swift test`
- [ ] `scripts/build_app.sh`
- [ ] GitHub Actions `Swift Build and Test` is green on the release commit
- [ ] App Artifact workflow produces an unsigned app artifact

## Manual App Checks

- [ ] KeyScout launches from `dist/KeyScout.app`
- [ ] KeyScout appears in the menu bar as `⌘?`
- [ ] Scan Frontmost App works against several common apps
- [ ] Generate Unused Shortcut returns a readable suggestion
- [ ] Export JSON writes a catalog file to Downloads
- [ ] App behavior remains local-only
- [ ] Accessibility permission copy is understandable

## Documentation Checks

- [ ] README status matches implemented behavior
- [ ] `docs/requirements.md` coverage matrix is current
- [ ] `docs/release.md` artifact instructions are current
- [ ] `CHANGELOG.md` Unreleased section is current
- [ ] `SECURITY.md` and `PRIVACY.md` cover any new permissions or storage
- [ ] Known limitations are explicit and not overstated

## Release Decision

- [ ] Unsigned build limitation is clear
- [ ] Signing/notarization status is clear
- [ ] Hardware/manual QA results are recorded
- [ ] GitHub release notes are prepared from `docs/v0.1-release-notes.md`
