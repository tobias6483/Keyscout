### Manual QA Notes

Manual QA on 2026-06-04 used `dist/KeyScout.app` from the local bundle script.

Passed:

- App launched as a menu bar accessory process.
- Menu bar item appeared as `⌘?`.
- Menu displayed current status text.
- Missing Accessibility permission state showed
  `Accessibility permission needed to scan app menus`.
- `Scan Frontmost App` did not report a successful scan while Accessibility
  permission was missing.
- `Open Accessibility Settings` opened System Settings and updated status to
  `Enable KeyScout in Accessibility settings`.
- `Open Shortcut List` opened the native `KeyScout Shortcuts` window.
- `Generate Unused Shortcut` returned the permission status while scanning was
  unavailable.
- `Export JSON` wrote `~/Downloads/keyscout-shortcuts.json`, and the exported
  file decoded as a valid empty KeyScout catalog.
- `Import Mapping JSON` imported `docs/examples/sample-catalog.json` and showed
  `Imported 2 shortcuts`.
- Invalid JSON import showed a readable failure status.
- No account, network, telemetry, or cloud behavior was observed during this
  pass.

Not completed in this pass:

- Accessibility permission was not granted, so successful frontmost-app scanning
  against real apps and populated Accessibility rows were not verified.
- Search and source filtering were not verified against Accessibility scan data.

Observed caveat:

- Importing a valid mapping file by itself does not populate the shortcut list
  when there is no latest scan whose app name or bundle identifier matches the
  imported mappings. This matches the current merge design, but it can make a
  successful import look empty until a matching app is scanned.

### Accessibility QA Notes

Accessibility QA on 2026-06-04 used an ad-hoc signed `dist/KeyScout.app`.

Passed:

- Accessibility permission remained active after relaunch once the app bundle
  was rebuilt with ad-hoc signing.
- `Scan Frontmost App` scanned Finder and reported 108 shortcuts.
- `Scan Frontmost App` scanned Safari and reported 93 shortcuts.
- `Scanned Shortcuts` submenu populated after a scan and showed overflow rows.
- `Open Shortcut List` showed 93 of 93 Safari shortcuts.
- Selecting a shortcut row showed conflict detail with app, command, and source.
- Search filtering reduced Safari rows from 93 to 0 for a no-match query.
- Source filtering showed 93 Accessibility rows, 0 Curated rows, and 93 rows
  again after returning to All Sources.
- `Generate Unused Shortcut` returned `Suggested unused shortcut: ⇧⌘A` in the
  tested state.
- `Export JSON` wrote a valid Finder catalog with 108 shortcuts to Downloads.
- `Import Mapping JSON` wrote 2 shortcuts to the local Application Support
  store, and the store still contained 2 shortcuts after app restart.
- `Quit` was enabled after retargeting it to `NSApp` and successfully exited
  KeyScout from the menu bar.

Observed caveats:

- The app bundle must be signed as a bundle, even with ad-hoc signing, before
  macOS Accessibility permission reliably binds to `dev.tobias.keyscout`.
- If the KeyScout shortcut list window is frontmost, `Generate Unused Shortcut`
  can rescan that frontmost context and replace the latest app catalog with an
  empty/non-useful catalog. Bring the target app frontmost before generating.
